# 🚀 CI&T - Do Prompt ao Agente

## 📌 Sobre o Projeto

Este repositório faz parte do meu aprendizado no curso **CI&T - Do Prompt ao Agente**, oferecido pela **DIO (Digital Innovation One)**.

O desafio consistia em criar dois **Agentes de IA** especializados:

1. **Agent 1 - Entrevistador de Carreira**: responsável por conduzir uma entrevista estruturada e sugerir as 3 melhores carreiras em tecnologia com base no perfil do usuário.
2. **Agent 2 - Planejador de Roadmap**: responsável por receber os dados do Agent 1 e gerar um plano de estudos completo e personalizado.

---

## 🧠 Contexto do Meu Perfil (Usado nos Agentes)

- **Formação**: Direito (Advogado)
- **Experiência em Tech**: Desenvolvedor recente em empresas + criador de um sistema jurídico próprio
- **Projeto atual**: Sistema jurídico com 50GB de decisões judiciais (web + dados + infraestrutura)
- **Disponibilidade**: 6 horas/semana para estudos
- **Objetivo**: Transição de carreira para área de dados/engenharia
- **Carreira escolhida**: Engenheiro de Dados (após análise do Agent 1)

---

## 🎯 Objetivo dos Arquivos

### `entrevista_agent1.md`
Contém a entrevista completa conduzida pelo **Agent 1**, com:
- 7 perguntas e respostas sobre interesses, experiência, disponibilidade, preferências e objetivos
- Análise do perfil e sugestão das 3 carreiras mais promissoras ranqueadas
- Transferência de informações para o Agent 2

### `roadmap_agent2.md`
Contém o plano de estudos gerado pelo **Agent 2**, com:
- Visão do dia a dia da carreira (Engenheiro de Dados)
- Mapa de skills (essenciais, complementares e ferramentas)
- Roadmap de 90 dias adaptado para 6h/semana
- Projeto de portfólio (Pipeline de Decisões Judiciais)
- Roteiro de 5 perguntas de entrevista com respostas modelo
- Trilha recomendada na DIO

---

## 💡 Por que Engenheiro de Dados?

O Agent 1 identificou que meu perfil tem um encaixe natural com Engenharia de Dados por causa de:
- Projeto real com 50GB de dados jurídicos (aumentando)
- Interesse em código + dados
- Experiência como advogado (validação de negócio + domínio único)
- Necessidade de estruturar pipelines confiáveis para decisões judiciais

## 🤔 Insight Interessante, mas com ressalva pessoal
Os agentes identificaram a Engenharia de Dados como a carreira tecnicamente mais alinhada com meu perfil atual (50GB de dados jurídicos + interesse em dados + código + infraestrutura). E de fato, é um insight interessante e factualmente correto.

No entanto, pessoalmente não concordo que esse seja o melhor caminho para mim.

Meu objetivo principal é ser Desenvolvedor (DEV) – full-stack ou backend – com um diferencial: gosto de mexer em todas as camadas (web, dados, infra), mas sempre com o foco no desenvolvimento de produtos, não na especialização pura em dados.

A Engenharia de Dados é uma área incrível e que certamente me beneficiaria, mas minha paixão está em construir sistemas completos, resolver problemas com código e criar produtos funcionais do início ao fim.

Resumo: Gosto de dados, mas não quero ser engenheiro de dados. Quero ser um DEV que também entende profundamente de dados.
